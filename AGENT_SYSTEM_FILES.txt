================================================================================
   QUALITY GATES (Enforced at Each Stage)
================================================================================

QG-COMMIT: On every git commit
  - Linting, formatting, type-checking, unit tests, secret detection
  - Action: Block commit if failed

QG-PR: On pull request
  - All QG-COMMIT + integration tests, E2E, coverage ≥85%, Lighthouse ≥90
  - Action: Block merge if failed

QG-DEPLOY: Before merge to main
  - All QG-PR + build artifact, Docker image, migrations validation
  - Action: Prevent deployment if failed

QG-MONITOR: After deployment (60 minutes)
  - Health checks, error rate <0.5%, performance <5% degradation
  - Action: Auto-rollback if failed
